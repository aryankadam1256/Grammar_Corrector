## Datasets Used

### 1. BEA-2019 (W&I+LOCNESS) — Primary Dataset

**Source:** HuggingFace (`juancavallotti/bea-19-corruption`)
**Purpose:** Training and evaluation
**Total:** 84,106 sentence pairs

| Split | Samples | Usage |
|-------|---------|-------|
| Train | 75,691 | Model training |
| Dev | 4,203 | Validation during training |
| Test | 4,206 | Final evaluation |

**Content:** Real errors from English language learners at different proficiency levels (A, B, C, Native). Sourced from the Write & Improve and LOCNESS corpora.

**Error Types Present:**
- Verb tense/form errors ("go" → "went")
- Subject-verb agreement ("they was" → "they were")
- Determiner errors ("a" missing, "the" extra)
- Preposition errors ("in" vs "on" vs "at")
- Spelling errors
- Punctuation errors
- Word order errors

**Format:** CSV with `source` (erroneous) and `target` (corrected) columns.

**Preprocessing:**
- Removed null/NaN rows
- Tokenized with T5 tokenizer (SentencePiece, max_length=256)
- Task prefix: `grammar: {source_text}`

---

### 2. CoNLL-2014 — Evaluation Benchmark

**Source:** CoNLL-2014 Shared Task
**Purpose:** Standard F0.5 evaluation benchmark
**Size:** 1,312 test sentences
**Format:** M2 format (parsed to CSV)

**Standard benchmark for GEC:** Nearly all published GEC papers report F0.5 on CoNLL-2014. Not used for training since it is only an evaluation set.

---

### 3. JFLEG — Fluency Evaluation

**Source:** GitHub (keisks/jfleg)
**Purpose:** Fluency evaluation using GLEU metric
**Size:** 747 test sentences with 4 references each

**Difference from CoNLL-2014:** JFLEG evaluates fluency (how natural the correction sounds) rather than just correctness. Multiple reference corrections allow for varied phrasings.

## Data Statistics

### BEA-2019 Token Statistics (T5 Tokenizer)
- Mean tokens per sample: ~50-80 tokens
- Max sequence length cap: 256 tokens
- Samples truncated: ~12%
- Padding overhead: ~35% (static padding to max_length)

### Error Distribution (BEA-2019 Train)
The dataset contains a natural distribution of learner errors, with verb errors and determiner errors being most common. The proficiency levels range from beginner (A) to native, providing diverse error patterns.

## Supplementary Data

### Synthetic GEC Dataset
**Generated by:** `scripts/generate_synthetic_gec_dataset.py`
**Size:** 8,418 samples
**Error Types:**
1. Subject-verb agreement
2. Verb tense
3. Article/determiner
4. Preposition
5. Spelling
6. Punctuation

**Purpose:** Supplement BEA-2019 training data with controlled error types.

### Data Augmentation
**Method:** Rule-based augmentation applied to BEA-2019 clean text
**Techniques:**
- Noise injection (15% error rate): spelling, verb form, article, punctuation errors
- Random deletion (10% of words)
- Random swap (10% of adjacent word pairs)

**Augmentation factor:** 2x (original + 2 synthetic copies per sample)

## Data Pipeline

```
Download (HuggingFace/GitHub)
    → Clean (dropna, dedup)
    → Split (80/10/10 or pre-split)
    → Augment (2x for training only)
    → Tokenize (T5 tokenizer, max_length=256)
    → DataLoader (batch_size=8, pin_memory=True)
```

**Files:** `src/data/download.py`, `src/data/preprocess.py`, `src/data/augmentation.py`
